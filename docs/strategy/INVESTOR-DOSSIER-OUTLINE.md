## 3. Valuation FRAMING (ranges + comparables, NOT a fixed number)

> **No fixed valuation is asserted.** What follows is *framing* — the comparables and the bracketing logic an investor conversation would reference. The actual number is set by the market and by Brandon, in a negotiation this document does not conduct.

**Comparable reference classes (for framing only):**

- **Pre-seed deep-tech / security infra (pre-revenue, pre-audit):** valuations are set far more by team, wedge, market timing, and non-dilutive validation than by revenue (there is none). Early open-core / dev-infra pre-seed rounds commonly land in a **low-single-digit-millions post-money** band, widening with traction signals.
- **Post-audit / post-first-pilots seed:** marks step up once the audit and first design-partner conversions de-risk the story; the step is **earned by clearing gates**, not assumed.
- **Government-validated deep tech (post-Phase I/II):** non-dilutive selection (SBIR/CSO, then Phase II) is itself a third-party validation signal that supports a higher mark — but only after it lands.

**Bracketing logic (assumptions, not a number):**

1. **Floor framing** — pre-audit, pre-revenue: justified by team + wedge + the non-dilutive stack already in motion; the lower the dilutive ask, the less the valuation conversation matters near-term (SAFE cap defers it).
2. **Step-up triggers** — audit complete; ≥1 live custody path; first paying pilots; a Phase I award. Each is a *de-risking event* that supports a higher mark; none is assumed.
3. **Ceiling framing** — only after multiple gates clear (audit + custody + FTO + Phase II + early ARR); explicitly the same "ceiling, not plan" caveat as the strategy doc's HIGH column.

**Discipline:** any number that appears in a live conversation is set there, with comparables in hand, by Brandon — **not** pre-committed here.
